RabbitMQ消息队列

发表于 2020-12-02 阅读次数：

RabbitMQ

消息队列可以发送 字符串、字节数组、序列化对象，需要对象实现序列化接口

直接使用docker启动容器,于是就可以愉快的使用了

docker run -d -p 5672:5672 -p 15672:15672 –name –restart=always rabbit rabbitmq

RabbitMQ构建基础生产者消费者模式

构建连接工厂

生产者

主要流程：

  1. 获取连接Connection
  2. 创建通道
    • parma1:队列名称
    • parma2:持久化，重启server队列不会消失
    • parma3:是否为独占队列。若队列只能被一个connect使用那么其他连接建立时抛出异常
    • parma4:队列使用完全是否自动删除（queue为空，并且没有连接时）
    • parma5:建立队列时的其他参数,Map类型
  3. 使用basicPublish进行广播

消费者

主要流程：

  1. 创建连接
  2. 创建通道
  3. 获取信息
  4. 消费者在消费消息时，要通过ack确认（即：消费者没有确认消息消费不返回ack）;

工作模式

一个生产者，数个消费者

两个消费者

订阅模式

发送者 发送消息到交换机

路由模式

通过信息来决定选择哪个路由
