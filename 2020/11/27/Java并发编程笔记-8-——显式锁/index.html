显式锁

Lock与ReentrantLock

ReentrantLock实现了lock接口，并提供了与Synchronize相同的互斥性和内存可见性

公平性

ReentrantLock提供了两种公平性的选择，创建一个非公平的锁（默认），或是‘一个公平的锁，

  • 公平：FIFO
  • 非公平：先请求的锁不一定先取得

默认使用非公平的原因：性能更高：在恢复一个被挂起的线程与这个线程正在开启运行的时间中存在严重的延迟。

内置的synchronize的锁也是非公平的
